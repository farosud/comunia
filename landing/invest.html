Investment thesis

We are not building another group chat bot. We are building software that helps places organize themselves.

01

Neighborhoods are the proving ground

Local communities already live inside chat. They coordinate events, favors, safety, vendors, introductions, and shared norms there. Comunia makes that coordination stateful and operational.

02

Memory compounds into governance

Once a system remembers preferences, relationships, recurring needs, and unresolved proposals, it stops acting like a bot and starts acting like a lightweight institution.

03

Agents can specialize by function

One agent should not do everything. Different agents can handle intake, matching, policy interpretation, event planning, local services, and long-term community memory.

04

Scale moves outward from real places

The same primitives that run a block, building, campus, or founder neighborhood can later power municipal workflows, civic programs, and larger coordinated communities with stricter rules.

Expansion path

First manage a neighborhood well. Then widen the jurisdiction.

Phase 1 Now

Neighborhood operator

Help small real-world communities coordinate events, intros, onboarding, and recurring needs across existing chat surfaces. This is the current product direction and wedge.

Phase 2 After seed

Multi-agent local administration

Add specialized agents for service requests, local commerce, safety alerts, volunteer coordination, policy memory, and subgroup routing. One shared community graph, many operators.

Phase 3 City-scale

Federated communities

Let many neighborhoods connect into a district or city layer while preserving local context. Communities keep autonomy, while shared agents coordinate resources, events, and signals across them.

Phase 4 Long term

Nation-state edge interfaces

The ambition is not to replace government. It is to create agent-mediated infrastructure for identity-aware coordination, public communication, program delivery, and policy execution at larger scales.

Agent network

More agents, narrower responsibilities, better coordination.

Intake agent

Turns messy chat into structured proposals, needs, issues, and requests.

Memory agent

Maintains persistent context for members, places, preferences, and unresolved threads.

Event agent

Plans gatherings, invitations, reminders, follow-up, and attendance patterns.

Matchmaking agent

Connects people to people, projects, opportunities, and micro-communities.

Service agent

Routes local service requests such as vendors, maintenance, supplies, and community support.

Policy agent

Tracks local rules, shared norms, approvals, and eventually jurisdiction-specific constraints.

Why now

LLMs make communities legible. Messaging apps make distribution free.

Communities already coordinate in chat

Behavior does not need to be retrained. The product meets people where the group already lives.

Agent quality is now good enough for operational tasks

Summarization, extraction, planning, triage, memory, and routing are finally reliable enough to be useful daily.

The infrastructure layer is still open

The category has many chat bots and very little software for real-world community operations at civic scale.
